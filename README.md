## Overview

**XSquads** is a modular ecosystem of specialized AI squads designed to tackle different categories of work through expert orchestration.

Instead of relying on a single generic assistant, this project organizes intelligence into **domain-specific squads**, each one composed of:

- an **orchestrator**
- a set of **specialist agents**
- **tasks** for repeatable execution
- **workflows** for multi-step delivery
- **checklists** for quality control
- **supporting data/config files** for consistency and scale

The result is a more structured, more strategic, and more production-oriented way to operate with AI.

---

## Philosophy

XSquads is built around a simple idea:

> Complex work deserves specialized thinking.

Each squad is designed to behave like a high-performance unit with clear roles, clear routing, and clear execution patterns.

That means:

- better task delegation
- more contextual outputs
- reusable structures
- scalable collaboration between agents
- stronger quality control
- less randomness, more system

---

## Core Idea

Traditional AI usage is often flat.

XSquads turns it into an **operational architecture**.

You don’t just ask for an answer.  
You activate the right squad for the right type of problem.

Examples:

- a **brand problem** goes to `brand-squad`
- a **conversion or persuasion problem** goes to `copy-squad`
- a **product/UI/design system problem** goes to `design-squad`
- a **growth/data problem** goes to `data-squad`
- a **security operation** goes to `cybersecurity`
- a **narrative or pitch problem** goes to `storytelling`
- a **paid acquisition challenge** goes to `traffic-masters`
- a **high-level strategic decision** goes to `advisory-board` or `c-level-squad`

## How It Works

The operating logic is simple:

1. **Choose the right squad**
2. **Activate the orchestrator**
3. **Run a diagnostic**
4. **Route the challenge to the best specialist(s)**
5. **Execute tasks or workflows**
6. **Validate output through review/checklists**

This creates a more disciplined AI workflow than prompt improvisation alone.

## Quick Start

### 1. Clone the repository

```bash
git clone https://github.com/leozitogs/xsquads.git
cd xsquads
```

### 2. Explore a squad

Choose a squad based on your objective and read its internal documentation.

Example:

```bash
cd brand-squad
```

### 3. Understand the squad components

Inside each squad, review:

- `README.md`
- `agents/`
- `tasks/`
- `workflows/`
- `squad.yaml`

### 4. Activate the right orchestrator

Each squad has its own activation model and specialist routing logic.

Examples of orchestrator-style activation found in the project:

```txt
@brand-chief
@copy-chief
@design-chief
@data-chief
@traffic-chief
@board-chair
```

## Design Principles Behind the Repository

This project is built on a few strong principles:

### 1. Modularity
Each squad is isolated enough to grow independently.

### 2. Specialization
Different domains require different types of reasoning.

### 3. Orchestration first
The system is designed around routing and coordination, not isolated output generation.

### 4. Repeatability
Tasks, workflows, and checklists create reusable execution patterns.

### 5. Scalability
New squads can be added without breaking the overall philosophy.
